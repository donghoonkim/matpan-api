# 프로젝트 진행 상황 공유 (2026-01-31)

금일 진행된 Matpan API 프로젝트의 주요 작업 내역을 공유합니다.

1. **관리자 보안 로그인 시스템 구축**:
   관리자 계정의 보안을 강화하기 위해 `bcrypt`를 이용한 비밀번호 암호화 저장 방식을 적용했습니다. 로그인 시 `JWT` 토큰을 발급하여 인증 상태를 유지하며, `Throttler`를 도입해 무차별 대입 공격(Brute-Force)을 방어하도록 구현했습니다.

2. **API 문서화 고도화 (Docs as Code)**:
   기존 Swagger UI에 더해, OpenAPI 3.0 표준을 준수하는 `api-docs.yaml` 명세서를 생성했습니다. 이를 통해 API 정의를 코드로 관리하고, 향후 ReDoc 등 다양한 도구와 연동할 수 있는 기반을 마련했습니다.

3. **리포지토리 보안 강화 (Public 전환 준비)**:
   외부 공개(Public)를 대비하여 `.gitignore` 규칙을 강화(로그, 인증서 차단)하고, 협업을 위한 안전한 환경 변수 템플릿인 `.env.example` 파일을 생성했습니다.

# 프로젝트 진행 상황 공유 (2026-01-28)

현재까지 진행된 Matpan API 프로젝트의 주요 작업 내역을 공유합니다.

1. **초기 구축 및 환경 설정**:
   NestJS 기반 프로젝트를 생성하고, API 서버 운영에 필요한 필수 패키지(`@nestjs/config`, `swagger`, `class-validator`)를 설치했습니다. 모든 API 엔드포인트는 전역적으로 `/api` 접두사를 사용하도록 설정되었으며, `/api-docs` 경로에서 자동으로 생성된 Swagger 문서를 확인할 수 있습니다.

2. **기능 구현**:
   게시글(Posts) 관리를 위한 샘플 리소스를 생성하고, 인메모리 방식의 CRUD API를 구현하여 기본적인 데이터 처리 로직과 유효성 검사 파이프라인(`ValidationPipe`) 동작을 검증했습니다.

3. **데이터베이스 연결**:
   Neon PostgreSQL 서비스와 연동하기 위해 `pg` 드라이버와 TypeORM을 도입했습니다. 보안을 위해 환경 변수(`.env`)로 접속 정보를 관리하며, SSL 모드를 활성화하여 안전하게 데이터베이스에 연결되도록 설정했습니다.

4. **문서화 및 보안**:
   프로젝트의 이해를 돕기 위해 `README.md`를 전면 한글화하고, 설치부터 테스트까지 가능한 '빠른 시작 가이드'를 추가했습니다. 또한, `.gitignore` 파일을 점검하여 민감한 정보가 저장소에 포함되지 않도록 조치했습니다.

현재 서버는 `npm run start` 명령어로 실행 가능하며, 로컬 환경에서 즉시 테스트할 수 있습니다.
